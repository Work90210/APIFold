# Design Context

## Target Audience
All segments: solo developers experimenting with MCP, engineering teams building AI products, and enterprise orgs managing API integrations at scale. The product serves different needs at different plan tiers.

## Brand Personality
Developer-focused. Clean, technical, Vercel/Linear-style — minimal, precise, trustworthy. The interface should feel like a professional engineering tool, not a consumer app.

## Design Principles
- **Precision over decoration** — every element earns its place
- **Data-first** — numbers and charts speak, chrome stays quiet
- **Trustworthy** — consistent, predictable, no surprises
- **Technical confidence** — monospace for data, clear hierarchy, no hand-holding

## Quality Bar
Flagship — this is a selling point. Pixel-perfect, every detail matters.

## Existing Design System
- **Colors**: Indigo brand (hue 243/275), tinted navy dark theme, OKLCH progressive enhancement
- **Tokens**: brand-400/500/600/700, status-success/warning/error/info, surface-1/2/3
- **Typography**: Plus Jakarta Sans headings, Inter body, fluid type scale with clamp()
- **Easing**: ease-out-expo (0.16, 1, 0.3, 1) for all transitions
- **Radii**: 0.5rem base, with xl and 2xl variants
- **Components**: Shadcn/Radix primitives via @apifold/ui package
